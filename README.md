# Telegram-бот записи на маникюр

Готовый асинхронный Telegram-бот на `python-telegram-bot 20.x` для записи на услуги с проверкой подписки на канал, управлением записями пользователя и админ-панелью.

## Функции

- Проверка подписки на канал при `/start` и перед записью.
- Пошаговая запись через `ConversationHandler`:
  - выбор услуги;
  - выбор даты через календарь месяца с навигацией;
  - выбор свободного времени.
- Ограничение: у пользователя может быть только одна активная запись одновременно.
- Просмотр и отмена своих записей (отмена доступна не позднее чем за 2 часа).
- Админ-команда `/admin`:
  - просмотр записей на дату;
  - удаление записи по ID.
- Фоновая архивация: прошедшие записи автоматически переводятся в статус `completed`.
- SQLite + SQLAlchemy 2.0 Async.
- Логирование в консоль и файл.
- Часовой пояс: `Europe/Moscow`.

## Структура проекта

- `bot.py` — точка входа.
- `config.py` — чтение конфигурации из `.env`.
- `database.py` — модели и методы работы с БД.
- `keyboards.py` — inline-клавиатуры.
- `handlers.py` — обработчики команд/кнопок и диалоги.
- `utils.py` — слоты, время, форматирование.
- `requirements.txt` — зависимости.
- `.env.example` — пример конфигурации.

## Установка и запуск

1. Установите Python 3.10+.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Создайте `.env` на основе примера:
   ```bash
   copy .env.example .env
   ```
4. Укажите в `.env`:
   - `BOT_TOKEN`
   - `CHANNEL_ID` (например, `@your_nail_channel`)
   - `ADMIN_ID`
5. Запустите:
   ```bash
   python bot.py
   ```

## Бесплатный хостинг (Render, webhook)

Для бесплатных платформ удобнее запускать бота в `webhook`-режиме.

1. Залейте проект в GitHub.
2. Создайте Web Service на [Render](https://render.com/):
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python bot.py`
3. В переменные окружения Render добавьте:
   - `BOT_TOKEN`
   - `CHANNEL_ID`
   - `ADMIN_ID`
   - `DATABASE_URL=sqlite+aiosqlite:///nail_bot.db`
   - `TIMEZONE=Europe/Moscow`
   - `LOG_FILE=bot.log`
   - `RUN_MODE=webhook`
   - `WEBHOOK_URL=https://<your-service>.onrender.com`
   - `WEBHOOK_PATH=/telegram-webhook`
   - `WEBAPP_HOST=0.0.0.0`
   - `PORT=10000` (или порт, который выдает платформа)

После деплоя бот сам поднимет webhook при запуске.

Примечание: на бесплатном тарифе Render сервис может "засыпать". Для постоянной доступности обычно нужен платный тариф или платформа с always-on free VM.

## Важно по подписке

Чтобы проверка подписки через `get_chat_member` работала корректно:

- бот должен быть добавлен в канал;
- у бота должны быть права на просмотр участников канала;
- в `.env` должен быть корректный `CHANNEL_ID`.

## Планировщик архивации

- Бот запускает периодическую задачу через `JobQueue`.
- Каждые 30 минут прошедшие активные записи переводятся в `completed`.
- Для работы `JobQueue` используется зависимость `APScheduler` (уже добавлена в `requirements.txt`).
